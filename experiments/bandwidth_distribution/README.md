## Bandwidth Distribution

In this argument, server and client spin up multiple QPs between each other and measure the throughput achieved through these parallel connections. By doing so, we can illustrate the fair bandwidth sharing realized through hardened MUXes and arbitration logic within BALBOA. The same bitstream as for the regular throughput evaluation experiments can be used, but this experiment requires specialized Coyote software found in this directory. It can be compiled with the same flow described for all other Coyote experiments, all paths in the CMake-scripts have beend updated accordingly. To build server and client respectively, one has to specify `-DINSTANCE = server_scaling` or `-DINSTANCE = client_scaling`.
